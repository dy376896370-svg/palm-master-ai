## 使用规则

- 本清单只用于人工检索、版本比对和逐字核验，不是原文内容库。
- 禁止让 AI 根据书名、关键词或现代解释补写原文。
- “已有原文”表示 `lib/palm-sources.ts` 已存在原文记录。
- “已核验”至少需要确认书名、版本、原文、章节或卷次，以及可复查链接。
- 中文古籍中的术语可能与现代“生命线、智慧线”等名称不完全对应。无法建立可靠对应时，保持未收录。
- 《冰鉴》《周易》并非现代掌纹识别手册。只有找到明确相关文本并能说明映射依据时才录入，否则仅作为文化背景，不作为掌纹原文。
- 录入公版资料时仍应使用必要的短引文，保留出处，不复制无关长段。

状态标记：

- `否 / 否`：尚未录入、尚未核验
- `是 / 是`：资料库已有记录，并已核验
- `待复核`：找到候选资料，但尚未完成版本和上下文核验

## 人工核验录入流程

1. 找到公共图书馆、国家数字资源库、Wikisource、Internet Archive 或 Project Gutenberg 等可复查版本。
2. 记录完整书名、作者或编者、版本年代、卷次或章节。
3. 对照扫描页与文本版，确认没有 OCR 错字、断句错误或现代改写。
4. 判断原文是否确实对应当前掌纹，而不是仅有相似的性格或命运主题。
5. 只选取展示所必需的短引文。
6. 编写现代解释或中文翻译时，不加入原文没有的断言。
7. 在代码评审中同时提交原文、出处链接、解释和核验备注。
8. 通过评审后再将“已有原文”和“已核验”改为“是”。

## 待解决的术语映射

- 古籍“三才纹”中的天纹、人纹、地纹与现代三大主线是否能逐项稳定对应。
- “玉柱纹”与现代事业线或命运线的映射边界。
- “财运线”是否应拆分为太阳线、成功线、水星线等不同西方概念。
- “妻妾纹”等历史术语如何在现代产品中避免性别偏见和决定论。
- 《冰鉴》《周易》应作为背景解释还是原典来源；默认应为背景解释，除非存在明确掌纹文本。
